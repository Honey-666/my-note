# 方法

## 1.方法的概述

* 方法:完成特殊功能的代码块
* 方法的好处:
  * 提高代码的复用性,提高开发效率
  * 减少代码的冗余,降低代码的耦合性

## 2.方法的格式

```java
修饰符 返回值类型 方法名 (形参列表:形参类型1 形参名1,形参类型2 形参名2,.....,形参类型n 形参名n) {
    方法体;
    return 返回值;
}
```

## 3.两个明确

* 明确方法的返回值类型
* 明确方法的形参列表

## 4.方法的调用

* 特点:不调用,不执行

* 分类:

  * 如果方法属于对象,方法没有static关键字
    * 对象名.方法名()
  * 如果方法属于类,方法有static关键字
    * 类名.方法名
  * 调用本类中的方法
    * 方法名();
    * 单独调用
      * 方法名()
    * 输出调用
      * System.out.println(方法名());
    * 赋值调用
      * 数据类型 变量名 = 方法名();

  * 当返回值类型为void的时候,只能选择单独调用
  * 当返回值类型不是void的时候,推荐使用赋值调用

## 5.void关键字

* void:方法没有返回值
* 特点:
  * 方法的调用方式只能使用单独调用
  * 方法内部的return可以省略不写

## 6.方法的重载

* 重载:在同一个类中或者子父类继承关系中出现了方法名相同,形参列表不同的一种现象
* 特点:
  * 方法名必须相同
  * 形参列表必须不同(必须至少满足以下一点)
    * 形参的个数不同
    * 形参的数据类型不同
    * 形参的数据类型顺序不同

## 7.方法的参数特点

* 形式参数(形参):定义在方法参数列表中参数

* 实际参数(实参):调用方法时小括号中的参数
* 当实参是基本数据类型的时候
  * 传递过去的是数据值
  * 方法形参的数据值的改变不影响实际参数的数据值

* 当实参是引用数据类型的时候
  * 传递过去的是内存地址值
  * 方法形参的内存地址值的改变不影响实际参数的内存地址值
  * 方法形参的内存地址值不改变,但改变形参内存地址值里面的数据值,影响实际参数的内存地址值里面的数据值
